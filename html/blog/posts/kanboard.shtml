How I'm using Kanboard.

Kanboard is a docker container for project management which utilises the "Kanban" project management system developed by the car manufacturer Toyota. It aims to have a visual "flow" of work from beginning to completion in order limit the amount of concurrent work-in-progress and present it visually.

Obviously you can use this for anything from maintaining software tasks that need to be done, or organising daily or weekly tasks. Me? I'm using it to track games i'm playing and have beaten and TV Show and movies I want to watch. Why? Mostly because for me, it's nice to create lists of these kinds of things and be able to see a visual increase in the "completed work", or in this case, games completed and media watched!

Below are screenshots of my current Kanboard.

I've divided it into 3 columns (or "swimlanes" as Kanboard calls them) for my TV/movies:

  1. Backlog - What I want to watch
  2. Watching - What I'm watching or will be watching very soon (in 3 days or less)
  3. Finished - What I've finished watching
